WHAT IS VOICEFORGE:
A SaaS platform where any business owner can:
  1. Sign in with Google or email OTP (no passwords)
  2. Describe their business in plain text
  3. Upload documents (PDF/DOCX/TXT) or scrape a URL for AI training
  4. Pick an AI voice for their agent
  5. Get a real phone number and deploy a live AI calling agent
  6. The agent makes and receives REAL phone calls — for sales,
     support, tech help, or marketing — powered by their knowledge base

TWO SEPARATE REPOS:
  voiceforge-api/  ← Express.js backend · always-on · Railway
  voiceforge-web/  ← Next.js 14 frontend · UI only · Vercel

TECH STACK (never deviate):
BACKEND (voiceforge-api/):
  express + cors + helmet + morgan + dotenv
  jsonwebtoken + bcryptjs (for OTP hashing)
  mongoose (MongoDB Atlas — singleton connection)
  zod (all input validation)
  nodemailer (OTP email via Gmail SMTP)
  multer (memoryStorage — files as Buffer, never to disk)
  @aws-sdk/client-s3 + @aws-sdk/s3-request-presigner (Cloudflare R2)
  langchain + @langchain/openai + @langchain/pinecone (RAG pipeline)
  @pinecone-database/pinecone (vector store)
  openai (GPT-4o-mini for agent reasoning + embeddings)
  cheerio (web scraping)
  setInterval in worker/index.ts (background doc ingestion loop)

FRONTEND (voiceforge-web/):
  next.js 14 App Router + typescript strict
  next-auth v5 (bridges to Express JWT — Google + OTP providers)
  tailwind css + shadcn/ui components
  21st.dev animated hero (landing page)
  zustand (useUserStore, useAgentStore, useCallStore)
  swr (data fetching and caching)
  axios (API calls to Express with JWT interceptor)
  sonner (toast notifications)
  react-dropzone (file uploads)

AUTH — SIMPLE, NO PASSWORDS:
  Option 1: Google OAuth → NextAuth handles → we verify Google token
            on Express → find/create User → return JWT
  Option 2: Email OTP → Express sends 6-digit code via Nodemailer
            → user enters code → we verify → return JWT
  NextAuth stores the Express JWT in a session cookie.
  Axios interceptor attaches it to every API call.

VOICE CALLING STACK (Phase 1 — simple):
  Vapi.ai = the ONLY voice service we need.
  - Provides real phone numbers (~$2 each from free credit)
  - Manages the actual PSTN phone call
  - Has built-in STT (speech → text)
  - Has built-in TTS (text → speech, multiple voice options)
  - Fires webhooks to our Express backend during calls
  - We give Vapi: a system prompt + our webhook URL
  - Vapi handles everything else

  NO Twilio code. NO ElevenLabs in Phase 1. NO Deepgram in Phase 1.
  Just Vapi + OpenAI GPT-4o-mini.

HOW A CALL WORKS (the core loop):
  User clicks "Call" → POST /api/calls/outbound
  → Express calls Vapi API → Vapi dials the number
  → Callee picks up → callee speaks
  → Vapi STT converts speech to text
  → Vapi fires POST /api/calls/webhook with the text
  → Express queries Pinecone (user's namespace) for relevant context
  → Returns context chunks to Vapi
  → Vapi feeds context + conversation to GPT-4o-mini
  → GPT-4o-mini generates reply
  → Vapi TTS speaks the reply to the callee
  → Loop until hangup
  → Vapi fires end-of-call webhook
  → Express saves transcript, deducts credits

CONTEXT FILE SYSTEM (each agent has one):
When an agent is created, GPT-4o-mini generates a structured JSON
"context file" from the user's business description:
  {
    agentName, role, businessName, businessDescription,
    tone, language, callObjective,
    doList: string[],       // specific things agent MUST do
    dontList: string[],     // things agent must NEVER say/do
    openingLine: string,    // how agent introduces itself
    fallbackMessage: string // what to say when it doesn't know
  }
This JSON + retrieved RAG chunks form the system prompt for every call.

DATA ISOLATION (multi-tenant):
  Pinecone namespace = userId.toString()
  Company A's docs NEVER appear in Company B's agent search.

CREDITS:
  Sign-up: +50 free | Doc upload: -5 each | Scrape: -2 each
  Outbound call: -3 per minute (ceil) | Inbound: -2 per minute
  Pre-flight check: user must have ≥5 credits to start a call
  Real deduction happens at end-of-call webhook

ABSOLUTE RULES:
  1. TypeScript strict — no `any` unless impossible to avoid
  2. Every API input validated with Zod before any DB operation
  3. All protected routes use requireAuth middleware
  4. Webhook route: NO requireAuth — verify x-vapi-secret header
  5. express.raw() for /webhook MUST be before express.json() in index.ts
  6. multer memoryStorage only — files never touch disk
  7. Pinecone queries always scoped to userId namespace
  8. Worker uses setInterval NOT while(true) — keeps event loop free
  9. Frontend only calls Express — never MongoDB or Pinecone directly
  10. ONE Next.js API route only: /api/auth/[...nextauth]

Create Nodemailer transporter using config.email (Gmail SMTP with App Password).

generateOtp(): string
  → crypto.randomInt(100000, 999999).toString()  ← 6 digits

sendOtpEmail(email: string, otp: string): Promise<void>
  Send via transporter. Subject: "Your VoiceForge login code: {otp}"
  HTML body: Simple clean card — large centered 6-digit code, "Expires in 10 minutes"
  Plain text fallback: "Code: {otp} — expires in 10 minutes"

createOtpRecord(email: string, otp: string): Promise<void>
  1. Delete any existing OtpRecord for this email (avoid duplicates)
  2. const otpHash = await bcrypt.hash(otp, 10)
  3. OtpRecord.create({ email, otpHash, expiresAt: new Date(Date.now() + 10 * 60 * 1000) })

verifyOtp(email: string, otp: string): Promise<boolean>
  1. OtpRecord.findOne({ email, used: false })
  2. If not found or expiresAt < now: return false
  3. const valid = await bcrypt.compare(otp, record.otpHash)
  4. If valid: record.used = true; await record.save(); return true
  5. return false

# AUTH-02 — Auth Routes

POST /auth/send-otp:
Validate body: { email: z.string().email() }
1. Simple rate limit check (use Map in memory: email → timestamps[])
   If >3 requests in 15 minutes: 429 { error: 'Too many OTP requests. Wait 15 minutes.' }
2. const otp = generateOtp()
3. await createOtpRecord(email, otp)
4. await sendOtpEmail(email, otp)
5. Return { success: true, message: 'Code sent to your email' }
On email send failure: 500 { error: 'Failed to send email. Check SMTP config.' }

POST /auth/verify-otp:
Validate: { email: z.string().email(), otp: z.string().regex(/^\d{6}$/) }
1. const valid = await verifyOtp(email, otp)
2. If false: 401 { error: 'Invalid or expired code' }
3. Find or create User: { email, provider: 'otp', name: email.split('@')[0] }
4. const token = signToken({ userId: user._id.toString(), email })
5. Return { token, user: { id, name, email, credits, plan } }

POST /auth/google:
Body: { idToken: string }  ← Google ID token from NextAuth on frontend
1. Verify via: fetch(`https://oauth2.googleapis.com/tokeninfo?id_token=${idToken}`)
2. If response not ok: 401 { error: 'Invalid Google token' }
3. Extract: email, name from tokeninfo response
4. Find or create User: { email, provider: 'google', name }
5. Return { token, user }

GET /auth/me (requireAuth):
User.findById(req.user.userId).select('-__v')
Return { user: { id, name, email, credits, plan } }

requireAuth(req, res, next):
  const header = req.headers.authorization
  if (!header?.startsWith('Bearer ')) return res.status(401).json({ error: 'Authentication required' })
  try {
    const payload = verifyToken(header.slice(7))
    req.user = { userId: payload.userId, email: payload.email }
    next()
  } catch { res.status(401).json({ error: 'Invalid or expired token' }) }

ingestDocument(docId: string): Promise<void>
  1. Find KnowledgeDoc by docId → set status='processing', save
  2. const url = await getPresignedUrl(doc.r2Key)
     const buffer = Buffer.from(await (await fetch(url)).arrayBuffer())
  3. Choose LangChain loader by doc.type:
     'pdf'  → new PDFLoader(new Blob([buffer]))
     'docx' → new DocxLoader(new Blob([buffer]))
     default → build loader from buffer.toString('utf8')
     const rawDocs = await loader.load()
  4. const splitter = new RecursiveCharacterTextSplitter({ chunkSize:1000, chunkOverlap:150 })
     const chunks = await splitter.splitDocuments(rawDocs)
     Add metadata to each: { docId, userId: doc.userId.toString() }
  5. const pc = new Pinecone({ apiKey: config.pinecone.apiKey })
     const index = pc.Index(config.pinecone.index)
     await PineconeStore.fromDocuments(
       chunks,
       new OpenAIEmbeddings({ model:'text-embedding-3-small', apiKey: config.openai.apiKey }),
       { pineconeIndex: index, namespace: doc.userId.toString(), textKey:'text' }
     )
  6. doc.status='ready'; doc.chunkCount=chunks.length; doc.pineconeNS=doc.userId.toString()
     await doc.save()
  On error: doc.status='error'; doc.errorMsg=err.message; await doc.save(); throw err

src/rag/retrieve.ts:
retrieveContext(userId: string, query: string, topK = 5): Promise<string[]>
  try {
    const pc = new Pinecone({ apiKey: config.pinecone.apiKey })
    const index = pc.Index(config.pinecone.index).namespace(userId)
    const emb = new OpenAIEmbeddings({ model:'text-embedding-3-small' })
    const vec = await emb.embedQuery(query)
    const res = await index.query({ vector: vec, topK, includeMetadata: true })
    return res.matches.filter(m => (m.score??0)>0.6).map(m=>m.metadata?.text as string).filter(Boolean)
  } catch(err: any) {
    if (err.message?.includes('Namespace')) return []  // namespace not yet created — ok
    throw err
  }

--- COMMON GOTCHAS ---
BE-1: "Cannot read property of X" in route
      → connectDB() not awaited in that handler

BE-2: Multer file is undefined / req.file is undefined
      → Frontend must set Content-Type: multipart/form-data
      → Don't manually set Content-Type with axios — let it set boundary automatically

BE-3: Vapi webhook body is {} or undefined
      → express.raw() is not registered BEFORE express.json() in src/index.ts
      → Check that the route path matches EXACTLY '/api/calls/webhook'

BE-4: CORS error in browser
      → FRONTEND_URL in .env doesn't exactly match the browser origin (including port)

BE-5: JWT missing on API calls / 401 everywhere
      → lib/auth.ts not storing backendToken in JWT callback
      → lib/api.ts interceptor not reading (session as any).backendToken

BE-6: Pinecone "Namespace not found" error
      → Not a real error. Namespace creates automatically on first upsert.
      → Return [] in retrieve.ts catch block when message includes 'Namespace'

BE-7: R2 upload fails / endpoint error
      → R2_ACCOUNT_ID must NOT have https:// prefix — just the alphanumeric ID

BE-8: OTP email not arriving
      → SMTP_PASS must be the 16-char Gmail App Password, NOT your Gmail login
      → Check Gmail's "Less secure app access" or use App Password in 2FA settings

BE-9: Vapi webhook not receiving events during calls
      → API_PUBLIC_URL must be publicly accessible (ngrok for local dev)
      → Update Vapi assistant serverUrl in Vapi dashboard when ngrok URL changes
      → Vapi dashboard → Assistants → your assistant → Server URL

BE-10: Worker not ingesting pending docs
       → Check KnowledgeDoc.status is exactly 'pending' (case-sensitive)
       → Check startWorker() is called in src/index.ts after connectDB()

FE-1: "Hooks cannot be called inside server component"
      → Add "use client" directive at the very top of the file

FE-2: Session is null in dashboard page
      → middleware.ts matcher not including that route path
      → Check exported config.matcher in middleware.ts

FE-3: api.agents.list() returns 401
      → backendToken not in session — check lib/auth.ts JWT callback stores it
      → Check lib/api.ts interceptor is reading the right session field
