CONTEXT How the Context File System Works
THE CORE INTELLIGENCE

The context file is generated once at agent creation time by reading all uploaded documents. The AI reads everything the user uploaded, compresses it into a compact, call-optimized JSON file, and stores it on the Agent document. During every call, this file is always in the system prompt — no regeneration, no latency.

1
User uploads documents (PDFs, DOCX, TXT, CSV, URLs)
Files go to Cloudflare R2. Text extracted and chunked by LangChain. Vectors stored in Pinecone (userId namespace). Status shows "Indexing..." → "Ready".
async · background worker
2
User clicks "Create Agent" — context file is generated HERE
Express downloads full text of all user's ready documents from R2. Concatenates everything (up to 80k chars). Sends to Featherless.ai LLM with a specific extraction prompt. LLM returns structured JSON — the compact knowledge file.
one-time · at agent creation
3
Context file stored on Agent document in MongoDB
The compact JSON (300-800 tokens) lives in Agent.knowledgeFile. It contains: business summary, key products/services, common Q&A pairs, important facts, what agent should/shouldn't say.
persistent · always available
4
Every call: context file + RAG chunks assembled into system prompt
Layer 1: Agent persona (name, role, do/don't list). Layer 2: Context file (always injected, covers most questions). Layer 3: Live RAG chunks (only for specific lookups the context file doesn't cover). 80% of questions answered by Layer 2 alone — no Pinecone hit needed.
per call · layers 1+2 always · layer 3 when needed
✓

Why this works for your platform: A dental clinic uploads a 30-page brochure. Context file extracts: crown price ₹15,000, hours Mon–Sat 9–7, 3 branches, insurance accepted, common procedures. Agent answers 95% of caller questions from this compact file. Only very specific queries (exact step in a procedure, a specific staff member's schedule) need to fetch from Pinecone. Result: fast, accurate, cheap calls.

CONTEXT FILE SYSTEM (THE CORE INTELLIGENCE):
Each agent has a pre-built compact JSON "knowledge file" generated ONCE at
agent creation time. Flow:
  1. User uploads docs → R2 + Pinecone chunks (background worker)
  2. User creates agent → we read all doc texts from R2
  3. Featherless LLM reads all texts → generates compact knowledgeFile JSON
  4. knowledgeFile stored on Agent document in MongoDB
  5. Every call: knowledgeFile injected into system prompt (no regeneration)
  6. PLUS: live RAG chunks from Pinecone for specific queries

knowledgeFile structure:
{
  businessSummary: string,
  keyProducts: [{ name, price?, description, features[] }],
  commonQA: [{ question, answer }],
  importantFacts: string[],  // hours, location, contact, policies
  escalationTriggers: string[]  // things agent should escalate to human
}

  const campaign = await Campaign.findById(campaignId)
  const agent = await Agent.findById(campaign.agentId)
  const user = await User.findById(campaign.userId)
  
  await Campaign.findByIdAndUpdate(campaignId, { status:'running', startedAt:new Date() })
  
  const contacts = await CsvContact.find({ campaignId, status:'pending' })
  
  for (const contact of contacts) {
    // Check credits before EACH call
    const freshUser = await User.findById(user._id).select('credits')
    if (freshUser.credits < 5) {
      await Campaign.findByIdAndUpdate(campaignId, { status:'paused' })
      console.log(`Campaign ${campaignId} paused — insufficient credits`)
      break
    }
    
    await CsvContact.findByIdAndUpdate(contact._id, { status:'calling' })
    
    // Build personalized message for this contact using their notes
    const personalizedMetadata = {
      userId: campaign.userId.toString(),
      agentId: campaign.agentId.toString(),
      campaignId: campaignId,
      csvContactId: contact._id.toString(),
      contactName: contact.name,
      contactNotes: contact.notes || ''
    }
    
    try {
      const vapiRes = await triggerOutboundCall(
        agent.vapiAgentId, contact.phone, personalizedMetadata
      )
      await CallLog.create({
        userId: campaign.userId, agentId: campaign.agentId,
        campaignId, csvContactId: contact._id,
        vapiCallId: vapiRes.id, direction:'outbound',
        toNumber: contact.phone, status:'initiated', startedAt: new Date()
      })
    } catch (err) {
      await CsvContact.findByIdAndUpdate(contact._id, { status:'failed' })
      await Campaign.findByIdAndUpdate(campaignId, { $inc: { failed:1, called:1 } })
    }
    
    // Wait 30 seconds between calls to avoid spam detection
    await new Promise(resolve => setTimeout(resolve, 30000))
  }
  
  await Campaign.findByIdAndUpdate(campaignId, { status:'completed', completedAt:new Date() })

retrieveContext(userId, query, topK=3): Promise<string[]>
  const embeddings = new GoogleGenerativeAIEmbeddings({
    apiKey: config.gemini.apiKey, model:'text-embedding-004'
  })
  const pc = new Pinecone({ apiKey:config.pinecone.apiKey })
  const index = pc.Index(config.pinecone.index).namespace(userId)
  const vec = await embeddings.embedQuery(query)
  const res = await index.query({ vector:vec, topK, includeMetadata:true })
  return res.matches.filter(m=>(m.score??0)>0.55).map(m=>m.metadata?.text as string).filter(Boolean)
